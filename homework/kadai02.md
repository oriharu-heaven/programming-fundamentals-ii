変数（へんすう）とは？

一言で言うと…

**変数とは「データを入れる箱」です。**

---

箱のイメージ

```
┌─────────┐
│   age   │  ← 箱の名前（変数名）
│         │
│   25    │  ← 箱の中身（値）
└─────────┘
```

プログラムの中で、数字や文字などのデータに **名前をつけて保存** しておく仕組みです。

---

具体例で理解しよう

```python
# 「age」という名前の箱に「25」を入れる
age = 25

# 「name」という名前の箱に「田中」を入れる
name = "田中"

# 箱の中身を使う
print(name + "さんは" + str(age) + "歳です。")
# → 田中さんは25歳です。
```

---

「変数」という名前の由来

| 漢字 | 意味 |
|------|------|
| **変** | 変わる・変化する |
| **数** | 数・データ |

つまり、**「変わることができるデータ」** という意味です。
箱の中身はいつでも入れ替えられます。

```python
age = 25   # 最初は25
age = 26   # 誕生日が来たら26に変更！
```

---

変数を使うメリット

**① 同じ値を何度も書かなくていい**
```python
tax_rate = 0.1  # 一箇所で管理

price1 = 1000 * tax_rate  # → 100
price2 = 5000 * tax_rate  # → 500
price3 = 8000 * tax_rate  # → 800
```

**② 後から簡単に変更できる**
```python
# 消費税が変わっても、ここだけ直せばOK！
tax_rate = 0.12
```

**③ コードが読みやすくなる**
```python
# 悪い例（数字だけだと意味不明）
result = 1000 * 0.1

# 良い例（変数名で意味が伝わる）
price = 1000
tax_rate = 0.1
result = price * tax_rate
```

---
まとめ

> 変数 ＝ **「名前のついたデータの箱」**
> - 好きな名前をつけられる
> - 中身はいつでも変更できる
> - プログラムをわかりやすくしてくれる

変数はプログラミングの**最も基本的な概念**の一つです。これを理解すれば、プログラミングの学習がぐっと楽になります！💡

変数（へんすう）のシンプルな定義

変数とは、コンピュータの中に用意する**「データを入れておく、名前のついた箱」**のことです。



この「箱」のイメージを構成する要素は以下の3つに分けられます。

* **箱そのもの：** コンピュータのメモリ上（RAM）に確保された記憶スペース
* **箱に貼るラベル（変数名）：** そのデータが何なのかを示す名前（例：`score`、`playerName`）
* **箱の中身（値）：** 実際に保存される数字や文字などのデータ（例：`100`、`"勇者"`）

なぜ変数を使うのか？（エンジニアの視点）

プロの現場で変数を使う理由は、単にデータを記憶させるためだけではありません。システムを「正しく、誰が読んでも分かりやすく」設計するためです。

1.  **データに意味を持たせるため（可読性の向上）：**
    プログラムの中に突然 `3.14` という数字が書かれているより、`pi = 3.14` と「箱」に名前がついている方が、後からコードを読む他のチームメンバーにも「これは円周率のデータだ」と一瞬で伝わります。
2.  **状況に合わせて中身を書き換えるため（可変性）：**
    例えばゲームで敵を倒して点数が増えた時、`score` という箱の中身を `0` から `100` へと入れ替えることができます。
3.  **同じデータを何度も使い回すため（再利用性）：**
    同じ数値を何度も直接書く代わりに変数名を使えば、もしその数値を変更したくなった場合、大元の箱の中身を1箇所変えるだけで、プログラム全体の計算結果に反映されます。

実際のコードでのイメージ

例えば、アプリやゲーム開発（SwiftやC#など）では、以下のように箱を用意してデータを管理します。

```swift
// 「playerName」という名前の箱を用意し、"勇者"という文字を入れる
var playerName = "勇者"

// 「score」という名前の箱を用意し、最初は 0 を入れておく
var score = 0
```

プログラミングを始めたばかりの中高生などに教える機会がある場合は、単なる「メモリ上の記憶領域」という専門用語を使うよりも、この「中身を自由に入れ替えられる、ラベル付きの箱」というメタファーを用いると、アプリ開発におけるデータの流れを直感的に掴んでもらいやすくなります。

変数の基本概念に加えて、「変数（中身が変わる箱）」と「定数（一度入れたら中身を変えられない南京錠付きの箱）」の使い分けの基準について、さらに掘り下げて解説しましょうか？
